\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}

Application modernization is a major challenge for organizations relying on outdated versions of programming languages, frameworks, and dependencies. Despite the availability of specialized tools, a significant part of software migration remains manual: project analysis, incompatibility identification, source-code transformation, build and test execution, followed by repeated error-correction cycles. This process increases delivery time, cost, and debugging effort.

This final-year engineering project, conducted at \textbf{CGI Technologies and Solutions Morocco}, focuses on the design of an \textbf{intelligent, extensible, and governed Migration Factory}. Its objective is to industrialize software migration by combining deterministic orchestration with the assistance of specialized agents and large language models.

The deterministic layer controls workflow execution: source and target environment configuration, project analysis, migration-path construction, isolated workspace preparation, transformation application, installation, build and test execution, as well as technical log and evidence collection. Large language models complement this process by deepening the analysis, enriching migration planning, interpreting execution results, and assisting the developer throughout the migration workflow.

When an installation, build, or test failure occurs, the platform constructs a repair context from the detected errors, affected files, and current pipeline state. A model then proposes a corrective change as a source-code diff, while a second review mechanism evaluates its relevance and associated risks. The developer remains in control and may approve, reject, or request a revised proposal before any change is applied inside an isolated environment.

The platform is organized around specialized migration profiles. The current work covers the progressive migration of \textbf{Java/Spring Boot} applications and a dedicated \textbf{Angular} workflow, from analysis and planning to transformations, builds, and validation. These two case studies demonstrate that the same orchestration, traceability, and governance foundation can be adapted to multiple technology ecosystems.

The intended outcome is to reduce manual effort, accelerate migration cycles, improve reproducibility, and make every decision auditable, while maintaining controlled collaboration between deterministic automation, large language model intelligence, and human expertise.
